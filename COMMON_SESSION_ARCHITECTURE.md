# Переход на общую сессию для всех камер NVR

## 🎯 Цель оптимизации

Переход от индивидуальных сессий для каждой камеры к **общей сессии** для всех камер одного NVR.

### Проблема предыдущей реализации:

```python
# БЫЛО (неэффективно):
Камера 1 → сессия A
Камера 2 → сессия B
Камера 3 → сессия C
...
Камера 16 → сессия P
# Итого: 16 сессий на один NVR
```

**Недостатки:**
- ❌ Быстро исчерпывает лимит сессий NVR (15-20)
- ❌ Высокая нагрузка на сервер аутентификации
- ❌ Сложное управление множеством сессий
- ❌ Риск race condition при обновлении

### Новая реализация:

```python
# СТАЛО (оптимально):
Все камеры NVR → ОДНА общая сессия X
# Итого: 1 сессия на один NVR
```

**Преимущества:**
- ✅ Экономит ресурсы NVR (1 сессия вместо N)
- ✅ Не исчерпывает лимит подключений
- ✅ Проще управление и мониторинг
- ✅ Меньше нагрузки на сеть и CPU

---

## 🔧 Внесенные изменения

### 1. Упрощена функция `get_camera_session()`

**Было:**
```python
def get_camera_session(cam, nvr_http, user, password):
    """Получить индивидуальную сессию для камеры"""
    cam_key = f"{cam['nvr_host']}:{cam['channel']}"
    with camera_sessions_lock:
        if cam_key in camera_sessions:
            return camera_sessions[cam_key]
    
    # Создаем индивидуальную сессию
    session = nvr_login(nvr_http, cam['nvr_host'], user, password, force_logout=False)
    ...
```

**Стало:**
```python
def get_camera_session(cam, nvr_http, user, password):
    """
    Получить ОБЩУЮ сессию NVR для камеры.
    Все камеры одного NVR используют одну и ту же сессию.
    """
    # Всегда возвращаем общую сессию NVR (одна на все камеры)
    session = get_session(cam['nvr_host'])
    
    if session:
        log(f"[auth:cam{cam['id']:02d}] Используем общую сессию NVR: {session[:16]}...")
    else:
        log(f"[auth:cam{cam['id']:02d}] Общая сессия NVR еще не создана, ожидаем...")
    
    return session
```

**Изменения:**
- ✅ Удалена логика создания индивидуальных сессий
- ✅ Всегда возвращается общая сессия из `nvr_session`
- ✅ Упрощено логирование
- ✅ Нет блокировок и словарей

---

### 2. Удалена функция `camera_session_refresh()`

**Причина удаления:**
- Функция обновляла индивидуальные сессии камер
- Теперь общая сессия обновляется через `session_keeper()`
- Отдельное обновление для камер больше не нужно

**Замена:**
```python
# session_keeper() обновляет общую сессию для ВСЕХ камер
def session_keeper(nvr_http, nvr_host, user, password):
    while True:
        session = nvr_login(nvr_http, nvr_host, user, password, force_logout=False)
        if session:
            is_valid = session_verify(nvr_http, nvr_host, f"sessionID={session}")
            if not is_valid:
                log(f"[auth:{nvr_host}] Сессия невалидна, повторная авторизация...")
        time.sleep(SESSION_INTERVAL)
```

---

### 3. Удалены неиспользуемые переменные

**Удалено:**
```python
# Индивидуальные сессии для каждой камеры (больше не нужны)
camera_sessions = {}
camera_sessions_lock = threading.Lock()
```

**Причина:**
- Все камеры используют общую сессию из `nvr_session`
- Словарь `camera_sessions` больше не заполняется
- Lock `camera_sessions_lock` больше не используется

---

### 4. Упрощена функция `shutdown()`

**Было:**
```python
def shutdown(sig,frame):
    # Закрываем все активные сессии NVR
    for nvr_host, session in nvr_session.items():
        ...
    
    # Очищаем индивидуальные сессии камер
    with camera_sessions_lock:
        camera_sessions.clear()
    
    # Останавливаем все процессы
    ...
```

**Стало:**
```python
def shutdown(sig,frame):
    # Закрываем все активные сессии NVR (одна на каждый NVR)
    for nvr_host, session in nvr_session.items():
        ...
    
    # Останавливаем все процессы
    ...
```

**Изменения:**
- ✅ Удалена очистка `camera_sessions` (словарь удален)
- ✅ Упрщена логика завершения работы
- ✅ Закрываются только общие сессии NVR

## 📊 Сравнение подходов

| Параметр | Индивидуальные сессии | Общая сессия |
|----------|----------------------|--------------|
| **Количество сессий** | N (по одной на камеру) | 1 (на весь NVR) |
| **Нагрузка на NVR** | Высокая | Низкая |
| **Риск превышения лимита** | Высокий (при 16+ камерах) | Минимальный |
| **Сложность кода** | Средняя | Простая |
| **Изоляция проблем** | Хорошая | Средняя |
| **Потребление памяти** | Высокое | Низкое |
| **Скорость подключения** | Медленнее (N авторизаций) | Быстрее (1 авторизация) |

## 🎯 Архитектура новой системы

### При запуске:

```
1. Для каждого NVR:
   ├─ nvr_login() → Создается ОДНА общая сессия
   └─ session_keeper() → Запускается поток обновления сессии

2. Для каждой камеры:
   ├─ capture_sub() → Запускается поток захвата видео
   └─ get_camera_session() → Возвращает общую сессию NVR
```

### Во время работы:

```
Камера 1 ─┐
Камера 2 ─┤
Камера 3 ─┼──→ Общая сессия NVR (обновляется каждые 5 минут)
...       │
Камера 16 ┘
```

### При обновлении сессии:

```
session_keeper():
  ├─ nvr_login(force_logout=False) → Новая сессия
  ├─ session_verify() → Проверка валидности
  └─ Обновление nvr_session[nvr_host]
  
Все камеры автоматически получают новую сессию через get_camera_session()
```

### При остановке:

```
shutdown():
  ├─ Для каждого NVR:
  │   └─ session_logout() → Закрытие общей сессии
  └─ Остановка всех процессов
```

## 📈 Преимущества новой архитектуры

### 1. Экономия ресурсов NVR

**До:**
```
16 камер × 1 сессия = 16 сессий на NVR
Лимит NVR: 15-20 сессий
Использование: 80-100% ⚠️
```

**После:**
```
16 камер × 1 общая сессия = 1 сессия на NVR
Лимит NVR: 15-20 сессий
Использование: 5-7% ✅
```

### 2. Упрощение кода

**Удалено:**
- ❌ Функция `camera_session_refresh()` (~20 строк)
- ❌ Переменная `camera_sessions` (словарь)
- ❌ Переменная `camera_sessions_lock` (lock)
- ❌ Логика управления индивидуальными сессиями

**Упрощено:**
- ✅ Функция `get_camera_session()` (с 20 до 10 строк)
- ✅ Функция `shutdown()` (удалена очистка словаря)

### 3. Повышение надежности

**До:**
- ⚠️ Race condition при обновлении сессий
- ⚠️ Риск превышения лимита NVR
- ⚠️ Сложная отладка множества сессий

- ✅ Нет race condition (одна сессия)
- ✅ Невозможно превысить лимит (1 сессия << 15-20)

**До:**
- Медленный старт (N авторизаций последовательно)

- Низкая нагрузка на CPU (1 поток обновления на NVR)
- Минимальный расход памяти (1 сессия на NVR)

## ⚠️ Важные замечания

### 1. Изоляция проблем

**Возможная проблема:**
Если общая сессия истекает или становится невалидной, **все камеры** теряют подключение одновременно.

**Решение:**
- `session_keeper()` проверяет сессию каждые 5 минут
- `session_verify()` обнаруживает проблемы до их проявления
- Автоматическая переавторизация при обнаружении проблем

### 2. Масштабируемость

**Текущая реализация поддерживает:**
- До 64 камер на одном NVR (лимит каналов)
- До 15-20 NVR одновременно (лимит сессий)
- Итого: ~960-1280 камер в системе

**Для больших систем:**
- Рассмотрите балансировку нагрузки между несколькими серверами
- Используйте кластеризацию NVR

### 3. Совместимость

**Проверено с:**
- HeroSpeed NVR 64ch
- Longse OEM прошивки
- API v2.x (login-capabilities / login)

**Требует тестирования:**
- Старые firmware (< v2.0)
- Другие производители (Hikvision, Dahua)
