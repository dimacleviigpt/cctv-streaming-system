## 🎯 Рекомендации

### **Вариант A: Использовать GStreamer для RTSP** ✅

**Преимущества:**
- ✅ Работает со всеми 17 каналами
- ✅ Поддерживает H.264 и H.265
- ✅ Стандартный протокол RTSP
- ✅ Не требует WebSocket авторизации

**Недостатки:**
- ⚠️ Требуется установка GStreamer Python bindings
- ⚠️ Нужно обработать оба кодека (H.264/H.265)
- ⚠️ Разные аудио кодеки (PCMU/PCMA)
- ⚠️ Нет PTZ управления через RTSP

**Реализация:**
```python
import gi
gi.require_version('Gst', '1.0')
from gi.repository import Gst

def create_rtsp_pipeline(channel_id):
    url = f"rtsp://admin:356594wasq@10.120.204.4:554/{channel_id}"
    
    pipeline_str = (
        f'rtspsrc location="{url}" latency=200 protocols=4 ! '
        f'application/x-rtp,media=video ! '
        f'rtph265depay ! h265parse ! avdec_h265 ! '  # Для H.265
        f'videoconvert ! appsink'
    )
    
    return Gst.parse_launch(pipeline_str)
```
