# Управление сессиями HeroSpeed NVR

## 🎯 Проблема

При остановке скрипта оставались "висящие" сессии на NVR, что могло приводить к:
- Исчерпанию лимита одновременных сессий на NVR
- Проблемам с повторным подключением
- Утечке ресурсов на стороне сервера

## ✅ Решения

### 1. Автоматическое закрытие старой сессии при создании новой

Функция [`nvr_login()`](file:///home/zavklub/cctvo/cctv_stream.py#L167-L219) теперь:
1. Проверяет наличие существующей сессии
2. Закрывает её через `session_logout()` перед созданием новой
3. Логирует процесс замены сессии

```python
def nvr_login(nvr_http, nvr_host, user, password):
    # Шаг 0: Закрытие старой сессии (если существует)
    old_session = get_session(nvr_host)
    if old_session:
        log(f"[auth:{nvr_host}] Закрытие старой сессии: {old_session[:16]}...")
        session_logout(nvr_http, nvr_host, f"sessionID={old_session}")
    
    # ... создание новой сессии
```

**Преимущества:**
- ✅ Предотвращает накопление сессий при периодической авторизации
- ✅ Освобождает ресурсы на NVR
- ✅ Четкое логирование замены сессий

---

### 2. Корректное завершение всех сессий при остановке скрипта

Функция [`shutdown()`](file:///home/zavklub/cctvo/cctv_stream.py#L977-L1003) теперь:
1. Перебирает все активные сессии NVR
2. Закрывает каждую через `session_logout()`
3. Очищает индивидуальные сессии камер
4. Только затем завершает процессы и выходит

```python
def shutdown(sig,frame):
    """Корректное завершение работы с закрытием всех сессий NVR"""
    log("Остановка... Закрытие всех сессий NVR...")
    
    # Закрываем все активные сессии NVR
    for nvr_host, session in nvr_session.items():
        if session:
            try:
                log(f"[auth:{nvr_host}] Закрытие сессии при остановке: {session[:16]}...")
                for nvr in NVR_HOSTS:
                    if nvr["nvr_host"] == nvr_host:
                        session_logout(nvr["nvr_http"], nvr_host, f"sessionID={session}")
                        break
            except Exception as e:
                log(f"[auth:{nvr_host}] Ошибка при закрытии сессии: {e}")
    
    # Очищаем индивидуальные сессии камер
    with camera_sessions_lock:
        camera_sessions.clear()
    
    # Останавливаем все процессы
    kill_all_subs(); kill_all_main()
    ...
```

**Преимущества:**
- ✅ Чистое завершение работы без "висящих" сессий
- ✅ Освобождение ресурсов на всех NVR
- ✅ Логирование процесса завершения каждой сессии
- ✅ Обработка ошибок (продолжает закрытие других сессий даже если одна не закрылась)

## 📊 Жизненный цикл сессии

### При запуске:
```
1. nvr_login() → Создание сессии
2. session_keeper() → Периодическое обновление (каждые 5 минут)
   ├─ Проверка heartbeat
   └─ При необходимости: nvr_login() (закрывает старую → создает новую)
```

### При остановке (Ctrl+C или SIGTERM):
```
1. shutdown() вызывается
2. Для каждой сессии NVR:
   ├─ session_logout() → Закрытие сессии на сервере
   └─ Логирование результата
3. Очистка индивидуальных сессий камер
4. Завершение процессов
5. Выход из программы
```

## ⚙️ Конфигурация

Параметры управления сессиями в [`config.json`](file:///home/zavklub/cctvo/config.json):

```json
{
  "monitor": {
    "session_interval": 300,  // Интервал обновления сессии (секунды)
    ...
  }
}
```

**Рекомендации:**
- `session_interval`: 300 секунд (5 минут) - оптимально для большинства NVR
- Если сессии истекают быстрее, уменьшите до 180 (3 минуты)
- Если NVR поддерживает долгие сессии, можно увеличить до 600 (10 минут)

## 🧪 Тестирование

1. **Запустите скрипт:**
   ```bash
   python cctv_stream.py
   ```

2. **Дождитесь успешной авторизации:**
   ```

   ```

3. **Остановите скрипт (Ctrl+C):**

1. Дождитесь первого обновления сессии (через 5 минут по умолчанию)

   [HH:MM:SS] [auth:192.168.1.100] Новая сессия: xyz789...

## 🛠️ Отладка

### Если сессии не закрываются:

1. **Проверьте доступность NVR:**
   ```bash
   ping 192.168.1.100
   curl http://192.168.1.100:80/api/session/heart-beat
   ```

2. **Проверьте права доступа:**
   - Убедитесь, что пользователь имеет права на logout
   - Некоторые старые firmware могут не поддерживать logout

3. **Проверьте таймауты:**
   - Функция `session_logout()` использует timeout=5 секунд
   - Если NVR медленно отвечает, увеличьте timeout

### Если накапливаются сессии:

1. **Перезагрузите NVR** для сброса всех сессий
2. **Уменьшите `session_interval`** в config.json
3. **Проверьте логи** на ошибки авторизации

## 📝 Примечания

1. **Logout опционален**: Из-за короткого таймаута сессий (несколько минут), NVR автоматически очищает неактивные сессии. Однако явное закрытие - лучшая практика.

2. **Обработка ошибок**: Если `session_logout()` не удается (NVR недоступен), скрипт продолжает работу и пытается закрыть другие сессии.

3. **Индивидуальные сессии камер**: Очищаются в памяти, но не требуют отдельного logout на NVR, так как используют ту же сессию, что и основной NVR.

4. **Daemon потоки**: Потоки `session_keeper` помечены как daemon, поэтому они автоматически завершаются при выходе из основного потока.
