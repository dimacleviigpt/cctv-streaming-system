# Анализ лимитов сессий HeroSpeed NVR 64ch

## 📊 Найденная информация

### Huawei NVR800 (аналогичная архитектура)

Согласно официальной документации Huawei NVR800:

**Лимиты одновременных подключений:**
- **Web интерфейс**: максимум **7 одновременных входов** с разных браузеров
- **API/SDK**: лимит не указан явно, но обычно не превышает Web лимит
- **Общий лимит**: ~8 одновременных пользователей (включая системные)

**Важные ограничения:**
1. Не поддерживается несколько вкладок одного браузера для одного NVR
2. Каждый браузер считается отдельным подключением
3. API подключения учитываются в общем лимите

## 🔍 Применение к HeroSpeed 64ch

HeroSpeed использует прошивку, аналогичную Longse и другим OEM производителям. На основе найденной информации:

### Предполагаемые лимиты HeroSpeed 64ch:

| Тип подключения | Ожидаемый лимит | Примечания |
|----------------|----------------|------------|
| **Web интерфейс** | 5-10 сессий | Зависит от firmware |
| **API сессии** | 10-20 сессий | Включая наши скрипты |
| **Всего одновременно** | ~15-30 сессий | Суммарный лимит |

### Почему это важно для нашего проекта?

В текущей реализации мы создаем **индивидуальные сессии для каждой камеры**:

```python
# Для 16 камер на одном NVR:
Камера 1 → сессия A
Камера 2 → сессия B
Камера 3 → сессия C
...
Камера 16 → сессия P
```

**Проблема:** Если лимит NVR = 15-20 сессий, то при 16+ камерах мы можем превысить лимит!

---

## ✅ Рекомендуемая стратегия

### Вариант 1: Общая сессия для всех камер (РЕКОМЕНДУЕТСЯ)

**Преимущества:**
- ✅ Использует только **1 сессию** на NVR вместо N сессий
- ✅ Не исчерпывает лимит подключений
- ✅ Меньше нагрузки на NVR
- ✅ Проще управление

**Реализация:**
```python
# Одна сессия на весь NVR
nvr_session = nvr_login(nvr_http, nvr_host, user, password)

# Все камеры используют эту сессию
for camera in cameras:
    session = nvr_session  # Используем общую сессию
    connect_camera(camera, session)
```

**Недостатки:**
- ⚠️ Если сессия истекает, все камеры теряют подключение
- ⚠️ Требуется надежный session_keeper

---

### Вариант 2: Индивидуальные сессии (ТЕКУЩАЯ РЕАЛИЗАЦИЯ)

**Преимущества:**
- ✅ Изоляция проблем (одна камера не влияет на другие)
- ✅ Независимое обновление сессий

**Недостатки:**
- ❌ Быстро исчерпывает лимит сессий NVR
- ❌ При 16+ камерах возможны отказы
- ❌ Высокая нагрузка на NVR

**Когда использовать:**
- Только если камер ≤ 10 на один NVR
- Только если NVR поддерживает 50+ сессий

---

## 🎯 Рекомендация для вашего проекта

Учитывая, что у вас **64-канальный NVR**, рекомендую:

### Стратегия: Гибридная

1. **Для суб-потоков (thumbnails):**
   - Использовать **общую сессию** для всех камер
   - Экономит ресурсы NVR
   - Достаточно для MJPEG стримов

2. **Для основных потоков (fullscreen):**
   - Можно использовать индивидуальные сессии
   - Но ограничить до 5-10 одновременных просмотров

3. **Мониторинг лимитов:**
   ```python
   # Отслеживать количество активных сессий
   active_sessions = len(camera_sessions)
   if active_sessions > MAX_SESSIONS_PER_NVR:
       log(f"[WARNING] Превышен лимит сессий: {active_sessions}")
   ```

## ⚠️ Важные выводы

1. **HeroSpeed 64ch вероятно поддерживает 15-20 одновременных сессий**
   - Это меньше, чем количество каналов (64)
   - Индивидуальные сессии для каждой камеры могут превысить лимит

2. **Текущая реализация с индивидуальными сессиями рискованна**
   - При 16+ камерах возможны проблемы
   - Лучше перейти на общую сессию

3. **Рекомендуется протестировать лимит вашего конкретного NVR**
   - Создать тестовый скрипт для проверки
   - Мониторить ошибки аутентификации

4. **Оптимальное решение: общая сессия + fallback**
   - По умолчанию использовать общую сессию
   - Создавать индивидуальную только при проблемах

            print(f"\nМаксимальное количество сессий: {i-1}")

## 📝 Заключение

На основе анализа документации аналогичных систем (Huawei NVR800):

- **Ожидаемый лимит HeroSpeed 64ch: 15-20 одновременных сессий**
