# Исправление проблемы с отображением только 3 камер

## 🐛 Проблема

После добавления функционала автоматического закрытия старых сессий, система начала показывать только первые 3 камеры вместо всех доступных.

## 🔍 Причина

**Race Condition (гонка условий)** при обновлении сессий:

1. Функция `nvr_login()` автоматически закрывала старую сессию перед созданием новой
2. Несколько камер (потоков) вызывали `nvr_login()` почти одновременно
3. Когда камера #4 вызывала `nvr_login()`, она закрывала сессию, которую использовали камеры #1-3
4. Камеры #1-3 теряли валидную сессию и переставали работать
5. Результат: работали только первые 3 камеры, которые успели получить сессию до её закрытия

### Пример проблемы:

```
Время T0: Камера 1 вызывает nvr_login() → создает сессию A
Время T1: Камера 2 вызывает nvr_login() → закрывает сессию A, создает сессию B
Время T2: Камера 3 вызывает nvr_login() → закрывает сессию B, создает сессию C
Время T3: Камера 4 вызывает nvr_login() → закрывает сессию C, создает сессию D
         ⚠️ Камеры 1-3 теперь используют невалидную сессию!
```

## ✅ Решение

Добавлен параметр `force_logout` в функцию `nvr_login()`:

```python
def nvr_login(nvr_http, nvr_host, user, password, force_logout=False):
    """
    Args:
        force_logout: Если True, закрывает старую сессию перед созданием новой.
                     Если False (по умолчанию), просто обновляет сессию без logout.
    """
    if force_logout:
        old_session = get_session(nvr_host)
        if old_session:
            session_logout(nvr_http, nvr_host, f"sessionID={old_session}")
    # ... остальной код авторизации
```

### Обновленные функции:

1. **`session_keeper()`** - использует `force_logout=False`
   - Просто обновляет сессию без закрытия старой
   - NVR сам управляет жизненным циклом сессий

2. **`camera_session_refresh()`** - использует `force_logout=False`
   - Обновляет индивидуальную сессию камеры
   - Не влияет на другие камеры

3. **`get_camera_session()`** - использует `force_logout=False`
   - Создает новую сессию для камеры при необходимости
   - Не закрывает существующие сессии

4. **`shutdown()`** - использует явный `session_logout()`
   - Закрывает все сессии при остановке скрипта
   - Единственное место, где logout действительно необходим

## 📊 Как это работает теперь

### При запуске:
```
Камера 1: nvr_login(force_logout=False) → сессия A
Камера 2: nvr_login(force_logout=False) → сессия B (новая, A остается активной)
Камера 3: nvr_login(force_logout=False) → сессия C (новая, A и B остаются активными)
Камера 4: nvr_login(force_logout=False) → сессия D (все предыдущие активны)
✅ Все камеры работают!
```

### При обновлении сессий (каждые 5 минут):
```
session_keeper: nvr_login(force_logout=False) → новая сессия E
                (старая сессия D остается активной до истечения таймаута)
✅ Никаких конфликтов!
```

### При остановке скрипта:
```
shutdown():
  ├─ session_logout(сессия A)
  ├─ session_logout(сессия B)
  ├─ session_logout(сессия C)
  └─ session_logout(сессия D)
✅ Все сессии корректно закрыты!
```

## 🎯 Преимущества решения

1. **Нет race condition** - несколько потоков могут безопасно вызывать `nvr_login()`
2. **Все камеры работают** - сессии не закрываются преждевременно
3. **NVR управляет сессиями** - старые сессии автоматически истекают по таймауту
4. **Чистое завершение** - `shutdown()` явно закрывает все сессии
5. **Гибкость** - при необходимости можно использовать `force_logout=True`

## 📝 Примечания

### Почему HeroSpeed позволяет множественные сессии?

HeroSpeed NVR поддерживает несколько одновременных сессий от одного пользователя. Каждая сессия имеет уникальный `sessionID` и независимый таймаут. Это стандартное поведение для современных NVR систем.

### Таймаут сессий

- Сессии автоматически истекают через несколько минут неактивности
- `session_keeper` обновляет сессии каждые 5 минут (настраивается в config.json)
- Явный logout не требуется для нормального функционирования

### Когда использовать force_logout=True?

Параметр `force_logout=True` следует использовать ТОЛЬКО в следующих случаях:
1. При явном запросе пользователя на выход из системы
2. При обнаружении проблем с сессией (например, после сбоя сети)
3. В функции `shutdown()` для чистого завершения работы
